# 今日のコンテンツ一括生成

`docs/content-plan-4weeks.md` を参照して、今日の日付に対応する以下を生成する。

- **午前X投稿**: テキスト + 画像ファイルパス
- **午後X投稿**: テキスト + 画像ファイルパス
- **SEO記事**: `content/articles/{slug}.mdx` に保存

投稿はTeppeiが自分のタイミングで行う。Claudeは生成・保存のみ。

---

## 手順

### 0. 事前チェック（重複・閉店の確認）

コンテンツ生成に入る前に、以下を**必ず**確認する。

#### 既存ドラフトの確認
`docs/x-drafts.md` の先頭付近を読み、**今日の日付のドラフトが既に存在しないか**確認する。
- 存在する場合 → 「既にドラフトがあります」とTeppeiに報告し、上書きするか確認してから進む
- 存在しない場合 → 続行

#### 施設の営業状態チェック
今日使う施設（午前・午後）について、`data/facilities.json` の `closedAt` フィールドを確認する。
- `closedAt` が設定されている → 閉店済み。スキップして代替施設を選定
- `closedAt` が null → 続行

#### 既存記事の確認
今日生成予定のSEO記事スラグが `content/articles/` に既に存在しないか、Globで確認する。
- 存在する → スキップして自動選定ルールで代替記事を決定

---

### 1. 今日のプラン確認

`docs/content-plan-4weeks.md` を読み込み、今日の日付に対応する行を抽出する。

```
今日の日付 → 該当週・行を特定
→ 午前: 施設ID or 5選テーマ + フックタイプ
→ 午後: 施設ID + 「知ってた？」フック
→ 記事: タイトル案 + 種別（エリア/テーマ/初心者/デート）
```

プランに対象日がない場合（4週間終了後など）は「プランの対象期間外です。新しいプランを作成しますか？」と伝えて停止。

---

### 2. 施設データ取得

施設IDが指定されている投稿について、`data/facilities.json` から該当施設を読み込む。

**5選まとめの日**（午前が「5選まとめ」と記載されている場合）:
→ Step 3-A へ

**施設紹介の日**（午前・午後ともに施設ID指定）:
→ Step 3-B へ

---

### 3-A. 5選まとめ投稿の生成（午前が5選の日）

`/x-list-post` スキルの手順に準じてテキストを生成する。

- テーマはプランの記載通り（例:「実は3,000円以下で使える個室サウナ5選」）
- `data/facilities.json` から条件に合う施設を5件選定
- 裏どり（公式サイト確認）は実施する
- 画像テンプレート `scripts/x-list-card.html` を編集してスクリーンショット撮影
- 生成した画像パスを記録

---

### 3-B. 施設紹介投稿の生成（午前・午後）

午前・午後それぞれの施設について **並列で** 以下を実施:

#### DeepResearch（各施設）

以下を並列で調査:
- 公式サイト（WebFetch）: 最新料金・営業時間・設備・こだわりポイント
- Web検索（WebSearch）: 「{施設名} 口コミ」「{施設名} レビュー」
- 予約サイト（WebFetch）: bookingUrl がある場合

データ差異があれば `data/facilities.json` を修正する。

#### 投稿文生成

**午前（実験フック）**: その週のフック型に沿って生成

| 週 | フック型 | 冒頭パターン |
|----|---------|------------|
| Week3（3/23-3/29） | 価格型 | 「実は{ぼかしエリア}に{特徴}の個室サウナがある」 |
| Week4（3/30-4/5） | ターゲット型 | 「{ターゲット}にこそ行ってほしい個室サウナが{ぼかしエリア}にある」 |
| Week5（4/6-4/12） | No.1型 | 「全国一{特徴}な個室サウナ、知ってる？」 |
| Week6（4/13-4/19） | 組み合わせ型 | 「{コンセプト}×サウナが{ぼかしエリア}にある」 |

**⚠️ エリア・料金のぼかしルール（3/30〜）:**
- 冒頭パターンの `{ぼかしエリア}` は「都内」「関西」「中国地方」「東北」等の広域名を使う
- 「銀座」「広島」「三軒茶屋」等の具体地名はNG（リプで初出しにする）
- 料金の具体額も本文に書かない（「2,200円」「3,500円」等はNG）
- 末尾の「で、肝心の料金とエリアなんだけど——」と矛盾させないこと

**午後（「知ってた？」固定）**: 冒頭を必ず「知ってた？」から始める

```
知ってた？{ぼかしたエリア名}に{施設の特徴・驚きポイント}があるって。

{DeepResearchから得た具体的な魅力 2行}

で、肝心の料金とエリアなんだけど——
```

**共通ルール（情報ギャップ方式 3/30〜）**:
- 280ウェイト以内（日本語1文字=2, ASCII=1, 絵文字=2, 改行=1）
- URLを本文に入れない
- 絵文字1-3個
- **料金（具体額）を本文に書かない**
- **具体的なエリア名を本文に書かない** → 「都内」「中国地方」「関西」「東北」等でぼかす（「銀座」「広島」「三軒茶屋」等はNG）
- 末尾は必ず「で、肝心の料金とエリアなんだけど——」で終わる（リプへの引き）
- `docs/character-guide.md` の口調に準拠

#### リプライ文

```
{施設名}
💰 {料金}
📍 {具体エリア・最寄り駅} 徒歩○分
https://www.saunako.jp/facilities/{id}?utm_source=x&utm_medium=social
```

#### 画像確認（ファイル存在 + 中身の目視確認を必ず両方やる）

`public/facilities/{id}-0.webp` 〜 `{id}-3.webp` の4枚が存在するか確認。
不足している場合は取得してから続行（`/x-post` スキルのStep 4手順に準拠）。

**⚠️ ファイルが存在しても中身が壊れている場合がある（白紙画像、無関係な風景写真など）。**
**必ず Read ツールで画像を目視確認し、以下の基準でチェックすること。**

**投稿に使う画像の選定基準:**

| 種類 | 判定 |
|------|------|
| サウナ室内部（木の壁・ストーン・照明） | ✅ 優先 |
| 水風呂（温度計・演出が見えるもの） | ✅ 優先 |
| バレルサウナ・サウナ小屋外観 | ✅ OK |
| 外観・エントランス | ✅ OK |
| 外気浴スペース・休憩室 | ✅ OK |
| 施設外の風景（ロケーションがわかるもの） | ✅ OK（1枚まで） |
| 人物の後ろ姿 | ✅ OK |
| 白紙・真っ白・壊れた画像 | ❌ NG → 別画像に差し替え |
| 施設と無関係な風景（橋、観光地など） | ❌ NG → 別画像に差し替え |
| 人物の顔・正面が写っている | ❌ NG → 別画像に差し替え |
| 施設の様子が全くわからない（料理・小物のみ等） | ❌ NG → 別画像に差し替え |

NGの画像がある場合は `/x-post` スキルのStep 4に準じて差し替え画像を取得してから投稿する。

### 4. SEO記事生成（1日2本）

記事を毎日2本生成する。**1本目はプラン指定、2本目は自動選定**。

#### エリアガイド記事の事前チェック（掲載基準 + 画像）

エリアガイド記事を書く前に、対象エリアの**全施設**について以下を確認する:

1. **掲載基準の確認**: CLAUDE.md の「個室サウナの定義と掲載基準」に照らして、各施設が掲載対象かチェック。特に:
   - テントサウナのみの施設 → 掲載しない
   - 宿泊者専用（日帰り不可）→ 掲載しない
   - 疑わしい場合はTeppeiに確認してから記事に含める
2. **画像の目視確認**: 記事に含める全施設の `-0.webp` を Read ツールで確認。白紙・無関係な画像があれば記事生成前に差し替える

#### 2本目の自動選定ルール

`content/articles/` に存在しないエリア記事を以下の優先順位で選ぶ:

1. `docs/content-plan-4weeks.md` の翌日以降のプラン記事（先取り）
2. 未作成のエリアガイド（`data/facilities.json` の prefecture 一覧から未作成のものを選定）
3. 優先度: 施設数が多い都道府県 → 検索ボリュームが高そうな都市圏 の順

#### 既存記事チェック

`content/articles/` に同名スラグが存在する場合はスキップして「既存記事あり: {slug}」と報告。
2本とも既存の場合は「本日分の記事はすべて作成済み」と報告して終了。

#### スラグの命名規則

```
エリア記事: {prefecture}-private-sauna-guide
テーマ記事: {テーマ}-private-sauna（例: couple-date-private-sauna）
```

#### 記事生成ルール

- **文体**: `docs/character-guide.md` 準拠（友達口調・敬語禁止・AIっぽい言い回し禁止）
- **文字数**: 800字以上
- **内部リンク**: 関連記事を1本以上
- `data/facilities.json` の該当施設データを参照して正確な情報を記載
