# 今日のコンテンツ一括生成

`docs/content-plan-4weeks.md` を参照して、今日の日付に対応する以下を生成する。

- **午前X投稿**: テキスト + 画像ファイルパス
- **午後X投稿**: テキスト + 画像ファイルパス
- **SEO記事**: `content/articles/{slug}.mdx` に保存

投稿はTeppeiが自分のタイミングで行う。Claudeは生成・保存のみ。

---

## 手順

### 1. 今日のプラン確認

`docs/content-plan-4weeks.md` を読み込み、今日の日付に対応する行を抽出する。

```
今日の日付 → 該当週・行を特定
→ 午前: 施設ID or 5選テーマ + フックタイプ
→ 午後: 施設ID + 「知ってた？」フック
→ 記事: タイトル案 + 種別（エリア/テーマ/初心者/デート）
```

プランに対象日がない場合（4週間終了後など）は「プランの対象期間外です。新しいプランを作成しますか？」と伝えて停止。

---

### 2. 施設データ取得

施設IDが指定されている投稿について、`data/facilities.json` から該当施設を読み込む。

**5選まとめの日**（午前が「5選まとめ」と記載されている場合）:
→ Step 3-A へ

**施設紹介の日**（午前・午後ともに施設ID指定）:
→ Step 3-B へ

---

### 3-A. 5選まとめ投稿の生成（午前が5選の日）

`/x-list-post` スキルの手順に準じてテキストを生成する。

- テーマはプランの記載通り（例:「実は3,000円以下で使える個室サウナ5選」）
- `data/facilities.json` から条件に合う施設を5件選定
- 裏どり（公式サイト確認）は実施する
- 画像テンプレート `scripts/x-list-card.html` を編集してスクリーンショット撮影
- 生成した画像パスを記録

---

### 3-B. 施設紹介投稿の生成（午前・午後）

午前・午後それぞれの施設について **並列で** 以下を実施:

#### DeepResearch（各施設）

以下を並列で調査:
- 公式サイト（WebFetch）: 最新料金・営業時間・設備・こだわりポイント
- Web検索（WebSearch）: 「{施設名} 口コミ」「{施設名} レビュー」
- 予約サイト（WebFetch）: bookingUrl がある場合

データ差異があれば `data/facilities.json` を修正する。

#### 投稿文生成

**午前（実験フック）**: その週のフック型に沿って生成

| 週 | フック型 | 冒頭パターン |
|----|---------|------------|
| Week3（3/23-3/29） | 価格型 | 「実は{地名}に{価格}の個室サウナがある」 |
| Week4（3/30-4/5） | ターゲット型 | 「{ターゲット}にこそ行ってほしい個室サウナが{地名}にある」 |
| Week5（4/6-4/12） | No.1型 | 「全国一{特徴}な個室サウナ、知ってる？」 |
| Week6（4/13-4/19） | 組み合わせ型 | 「{コンセプト}×サウナが{地名}にある」 |

**午後（「知ってた？」固定）**: 冒頭を必ず「知ってた？」から始める

```
知ってた？{地名}に{施設の特徴・驚きポイント}があるって。

{DeepResearchから得た具体的な魅力 2行}

📍{エリア名}
#個室サウナ #サウナ子
```

**共通ルール**:
- 280ウェイト以内（日本語1文字=2, ASCII=1, 絵文字=2, 改行=1）
- URLを本文に入れない
- 絵文字1-3個
- 価格の具体的な金額は書かない
- `docs/character-guide.md` の口調に準拠

#### リプライ文

```
{施設名}の詳細はここからチェックしてみてね👇
saunako.jp/facilities/{id}
```

#### 画像確認

`public/facilities/{id}-0.webp` 〜 `{id}-3.webp` の4枚が存在するか確認。
不足している場合は取得してから続行（`/x-post` スキルのStep 4手順に準拠）。

**投稿に使う画像の選定基準（4枚ある場合も中身を確認）:**

| 種類 | 判定 |
|------|------|
| サウナ室内部（木の壁・ストーン・照明） | ✅ 優先 |
| 水風呂（温度計・演出が見えるもの） | ✅ 優先 |
| 外観・エントランス | ✅ OK |
| 外気浴スペース・休憩室 | ✅ OK |
| 施設外の風景（ロケーションがわかるもの） | ✅ OK |
| 人物の後ろ姿 | ✅ OK |
| 人物の顔・正面が写っている | ❌ NG → 別画像に差し替え |
| 施設の様子が全くわからない（料理・小物のみ等） | ❌ NG → 別画像に差し替え |

NGの画像がある場合は `/x-post` スキルのStep 4に準じて差し替え画像を取得してから投稿する。

---

### 4. SEO記事生成（1日2本）

記事を毎日2本生成する。**1本目はプラン指定、2本目は自動選定**。

#### 2本目の自動選定ルール

`content/articles/` に存在しないエリア記事を以下の優先順位で選ぶ:

1. `docs/content-plan-4weeks.md` の翌日以降のプラン記事（先取り）
2. 未作成のエリアガイド（`data/facilities.json` の prefecture 一覧から未作成のものを選定）
3. 優先度: 施設数が多い都道府県 → 検索ボリュームが高そうな都市圏 の順

#### 既存記事チェック

`content/articles/` に同名スラグが存在する場合はスキップして「既存記事あり: {slug}」と報告。
2本とも既存の場合は「本日分の記事はすべて作成済み」と報告して終了。

#### スラグの命名規則

```
エリア記事: {prefecture}-private-sauna-guide
テーマ記事: {テーマ}-private-sauna（例: couple-date-private-sauna）
```

#### エリアガイド記事フォーマット

```mdx
---
title: "{都道府県}の個室サウナ完全ガイド【全{N}施設】"
description: "{都道府県}の個室サウナ{N}施設を完全網羅。{主要都市}を中心に、料金・アクセス・特徴を徹底比較。"
category: "area-guide"
tags: ["{都道府県}", "個室サウナ", "エリアガイド"]
publishedAt: "{今日の日付}"
updatedAt: "{今日の日付}"
thumbnail: "/facilities/{その地域の代表施設id}-0.webp"
author: "サウナ子"
facilityIds: [{地域の全施設ID羅列}]
---

## {都道府県}に個室サウナ、{N}施設もあるって知ってた？

{2-3行のリード。サウナ子口調。全国に個室サウナが広がってることに触れる}

## {都道府県}の個室サウナ全{N}施設

**【重要】各施設は必ず以下の順番で記載すること。画像を省略しない。**

![{施設名}](/facilities/{id}-0.webp)

<FacilityCard id={id} />

---

（全施設繰り返し）

---

## {主要都市}エリアから選ぶ

### {市区町村1}

{2-3行の解説}

### {市区町村2}（複数エリアがある場合）

{2-3行の解説}

---

## 選び方のポイント

{箇条書き3点。初めての人向けに}

---

## まとめ

{2-3行。サウナ子らしく締める}

関連: [{内部リンク先記事タイトル}]({内部リンクURL})

---

<p className="text-xs text-gray-400 mt-8">※ 画像は各施設の公式サイトより引用しています。掲載に問題がある場合はお問い合わせください。</p>
```

#### テーマ記事フォーマット

```mdx
---
title: "{テーマタイトル}"
description: "{テーマの説明。検索意図に沿って}"
category: "column"
tags: ["{タグ1}", "{タグ2}", "個室サウナ"]
publishedAt: "{今日の日付}"
updatedAt: "{今日の日付}"
thumbnail: "/facilities/{代表施設id}-0.webp"
author: "サウナ子"
facilityIds: [{関連施設ID}]
---

## {フック見出し}

{リード文。サウナ子口調。テーマの文脈に引き込む}

## {本題セクション1}

{800字以上になるよう具体的に記述}

<FacilityCard id={id} />

## {本題セクション2}

## まとめ

関連: [{内部リンク}]

<p className="text-xs text-gray-400 mt-8">※ ...</p>
```

#### 記事生成ルール

- **文体**: `docs/character-guide.md` 準拠（友達口調・敬語禁止・AIっぽい言い回し禁止）
- **文字数**: 800字以上
- **内部リンク**: 関連記事を1本以上
- `data/facilities.json` の該当施設データを参照して正確な情報を記載

---

### 5. ビルド確認

データ差異修正 or 記事新規作成があった場合:

```bash
npm run build && npm run lint
```

エラーがあれば修正してから続行。

## 注意事項

- 投稿はTeppeiが行う。Claudeは投稿しない
- データ差異修正 → ビルド確認 → コミット & プッシュは自動で行う
- 閉店・休業が判明した場合は該当投稿をスキップして報告
- 5選まとめの日は午前の生成に時間がかかる旨を事前に伝える
