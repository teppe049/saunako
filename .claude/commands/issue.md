# Issue作成

GitHub Issueを効率的に作成する。

## 入力
- ユーザーからの簡単な説明（引数 or 対話で取得）

## 手順

1. 説明からIssueの種類を判定する
   - `bug`: バグ報告
   - `feature`: 新機能
   - `improvement`: 改善
   - `chore`: メンテナンス

2. 以下の構造でIssue本文を作成する
   ```
   ## 概要
   [1-2文で簡潔に]

   ## 詳細
   [必要に応じて背景・再現手順・期待動作]

   ## 完了条件
   - [ ] [具体的なチェックリスト]
   ```

3. GitHub MCPでIssueを作成する
   - owner: `teppe049`
   - repo: 現在のリポジトリ名（`git remote` で確認）
   - ラベル: 種類に応じて付与（存在するラベルのみ使用）

4. 作成したIssueのURLを報告する
