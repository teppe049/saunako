# Ship - ビルド→lint→コミット→プッシュ

以下の手順を順番に実行する。途中でエラーが出たら自動修正を試み、修正後に再実行する（最大3回）。

## 手順

1. **型チェック**: `npx tsc --noEmit`
   - エラーがあれば修正して再実行

2. **Lint**: `npm run lint`
   - エラーがあれば修正して再実行

3. **ビルド**: `npm run build`
   - エラーがあれば修正して再実行

4. **コミット**: 変更内容を分析して適切なコミットメッセージを生成し、コミットする
   - conventional commits形式（feat:, fix:, chore: など）

5. **プッシュ**: `git push`

## ルール
- 各ステップでエラーが出たら、エラー内容を分析して自動修正を試みる
- 3回修正しても通らない場合は、エラー内容を報告して停止する
- プッシュ前に変更ファイル一覧を表示して確認を取る
