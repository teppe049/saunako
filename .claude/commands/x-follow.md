# X フォロー戦略（日次ルーティン）

サウナ子（@saunako_jp）の日次いいねルーティンを実行する。

引数: $ARGUMENTS（`巡回` / `補充` / `棚卸し` / 省略時は `巡回`）

## ツール選択

**Playwright MCP を標準で使う**（X API はレートリミットが厳しすぎて実用不可）。

- プロフィール確認: `browser_navigate` → `browser_snapshot` で `x.com/{handle}` を開く
- 検索: `browser_navigate` で `x.com/search?q={query}&f=live` を開く → `browser_snapshot`
- フォロワー確認: `x.com/saunako_jp/followers` を開く
- ボタン操作: `browser_evaluate` で `element.click()` を使う（X.comはオーバーレイ問題があるため）

## モード

| モード | 説明 | 頻度 |
|--------|------|------|
| `巡回` | ターゲット+施設の新着チェック→いいね候補一覧 | 毎日 |
| `補充` | `#サ活` 等で新ターゲットを発掘しリストに追加 | 随時（15人未満の時） |
| `棚卸し` | 3日無反応の入れ替え+リスト整理 | 週1 |

1. `docs/x-like-strategy.md` の「現在のターゲットリスト」セクションを読み取る

3. `docs/x-like-strategy.md` の「実績ログ」セクションを読み取る

## モード: 巡回（デフォルト）

### Step 1. ターゲットリストの新着チェック

ターゲットリスト全員のプロフィールを **Playwright で順番に** 開いて最新投稿を確認する。

各ターゲットについて:
1. `browser_navigate` で `x.com/{handle}` を開く
2. `browser_snapshot` で最新投稿の内容・日時・いいね数を取得
3. 次の人へ進む（1人あたり navigate + snapshot の2ステップ）

**効率化**: snapshotで読み取れる情報（投稿内容・日時・いいね数・Liked済みか）をメモしながら進む。

**各投稿を以下の基準で判定**:

| 投稿の鮮度 | 内容 | 判定 |
|-----------|------|------|
| 24時間以内 | サウナ系 | ✅ 最優先 |
| 24時間以内 | 日常（ご飯・趣味等） | ✅ いいね少ないなら目立つ |
| 2-3日前 | サウナ系 | ✅ OK |
| 2-3日前 | 日常 | ❌ スキップ |
| 1週間以上前 | — | ❌ スキップ → 入れ替え候補マーク |

**いいねNG**:
- 愚痴・ネガティブ投稿
- リプ会話の途中
- RT/引用のみ
- いいね30+の投稿
- @sauna_ikitai の投稿

### Step 2. 施設公式アカウントの新着チェック

アクティブ施設アカウントを **Playwright で順番に** 開いて最新投稿を確認:

**毎日チェック**: @saunakuveta, @sauna_ladle, @totonoi_plus, @loylyland_info, @a_sauna_sapporo
**写真のみ**: @TAKANAWA_SAUNAS, @SHIBUYA_SAUNAS, @RED_ESAUNA

各施設1投稿ずつ、合計3-5いいねが目安。

### Step 3. 結果出力

以下のフォーマットで出力:

```
## 今日のいいね巡回レポート

### ターゲットリスト（{n}人）

| # | アカウント | 投稿内容（要約） | 鮮度 | 判定 | 理由 |
|---|-----------|----------------|------|:----:|------|
| 1 | @xxx | サウナ後のビール最高 | 3h前 | ✅ | サウナ系・24h以内 |
| 2 | @yyy | 投稿なし | — | ⏭️ | 1週間以上前 → 入れ替え候補 |

→ いいね候補: {n}件 / スキップ: {n}件 / 入れ替え候補: {n}人

### 施設公式

| 施設 | 投稿内容（要約） | 判定 |
|------|----------------|:----:|
| @saunakuveta | 新メニュー紹介 | ✅ |

→ いいね候補: {n}件

### アクション
- ✅ いいねする: {合計n}件
- ⚠️ リスト入れ替え候補: {n}人（→ `/x-follow 補充` で補充）

いいね対象のURLリストを出力するので、手動でいいねしてください。
```

### Step 4. いいね対象URL出力

いいね候補の投稿URLをリストで出力する（X MCPにいいね機能がないため手動）。

### Step 5. 実績ログ更新

`docs/x-like-strategy.md` の「実績ログ」テーブルに今日の結果を追記:

```
| {日付} | {ターゲットいいね件数} | {施設いいね件数} | {気づき} |
```

## モード: 補充

ターゲットリストが15人未満の場合、新しい候補を発掘する。

### Step 1. 現在のリスト確認

`docs/x-like-strategy.md` のターゲットリストを読み取り、不足人数を計算（目標15人）。

### Step 2. 候補検索（Playwright）

以下のキーワードで **Playwright で順番に検索**:

1. `x.com/search?q=%23サ活&f=live` — コア層
2. `x.com/search?q=サウナ%20行ってきた&f=live` — 一般ユーザー
3. `x.com/search?q=%23ととのう&f=live` — 熱量高い層

各検索結果から投稿者のプロフィールを開いて確認する。

### Step 3. 候補選定

プロフィールページで以下を確認:

| 条件 | 確認方法 |
|------|---------|
| フォロワー 100-2,000 | プロフィールの Followers 数 |
| bioにサウナ・サ活・ととのう | プロフィールの bio テキスト |
| 1週間以内に投稿あり | 最新投稿の日時 |
| 投稿のいいね 0-3 | 投稿のいいね数 |
| 写真付きサ活報告あり | 投稿に Image があるか |

**除外**:
- 既にリストにいる人
- @sauna_ikitai（競合）
- 施設公式アカウント
- bot / スパム
- フォロワー 100 未満（小さすぎ）

### Step 4. 結果出力

```
## ターゲット候補（{不足数}人分）

| # | アカウント | フォロワー | プロフ要約 | 直近投稿 | おすすめ理由 |
|---|-----------|-----------|-----------|---------|------------|
| 1 | @new_user | 340 | サウナ×ランニング | 昨日サ活報告 | いいね2、反応期待◎ |

リストに追加する？（「追加して」で実行）
```

### Step 5. リスト更新（ユーザー指示後）

ユーザーが承認したら:
1. `docs/x-like-strategy.md` のターゲットリストテーブルに追加
2. `docs/x-community-guide.md` の「今気になっている人」リストにも追加
3. 開始日に今日の日付を記入

## モード: 棚卸し

週1回のリスト整理。

### Step 1. フォロワーページで反応確認

1. `x.com/saunako_jp/followers` を開いて全フォロワーを確認
2. ターゲットリストの中にフォロワーがいるかチェック

### Step 2. 各ターゲットのプロフィール確認

開始日から3日以上経過しているターゲットについて:
1. `x.com/{handle}` を開く
2. 最新投稿のいいね数を確認（いいね30超が常態化していたら入れ替え候補）
3. プロフィールに「Follows you」表示があるかチェック

### Step 3. 判定

| 状態 | アクション |
|------|----------|
| フォローしてくれた | 卒業（リストから外す。普通に交流継続） |
| いいね返しあり | 継続（効果あり） |
| 3日以上反応なし | 入れ替え候補 |
| いいね30超が常態化 | 入れ替え（埋もれるので非効率） |

### Step 4. 結果出力

```
## ターゲットリスト棚卸し

| アカウント | 開始日 | 日数 | 反応 | 判定 |
|-----------|--------|------|------|------|
| @xxx | 3/12 | 5日 | いいね返し2回 | 継続 |
| @yyy | 3/12 | 5日 | なし | 入れ替え |
| @zzz | 3/13 | 4日 | フォロー済み | 卒業 |

→ 継続: {n}人 / 入れ替え: {n}人 / 卒業: {n}人
→ 補充が必要: {n}人（→ `/x-follow 補充` で発掘）

反映する？
```

### Step 5. リスト更新（ユーザー指示後）

1. `docs/x-like-strategy.md` から入れ替え対象を削除、卒業者にマーク
2. `docs/x-community-guide.md` も同期更新
3. 補充が必要なら `/x-follow 補充` の実行を提案

## 注意事項

- 1人1日1-2いいねまで（スパム防止）
- フォロー上限: 40/day、フォロー/フォロワー比率 2:1 以下
- @sauna_ikitai の投稿には絶対にいいねしない（競合）
- docs の2ファイル（`x-like-strategy.md` と `x-community-guide.md`）は常に同期を保つ
- X API (`search_tweets`) はレートリミットが厳しいため使わない。Playwright MCP で統一する
