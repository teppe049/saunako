# X 画像リスト投稿（○選まとめ）

保存されるまとめ投稿を作り、プロフィール訪問とサイト流入につなげる。

## 引数

テーマ（例: 「東京で初デート向きの個室サウナ5選」「5,000円以下でカップルOKの個室サウナ」）

## 手順

### 1. テーマ確定

引数がない場合、以下から提案してユーザーに選んでもらう:
- 投稿カレンダー（`docs/x-drafts.md`）の次のテーマ
- Search Console で imp が多いキーワードに関連するテーマ
- X戦略（`docs/x-strategy.md`）の4カテゴリから

テーマには必ず以下を含める:
- **誰向けか**（デート/初心者/女性/コスパ重視 等）
- **何を選んだか**（個室サウナ ○選）
- **エリアや条件**（東京/大阪/5,000円以下 等）

### 2. 施設選定

`data/facilities.json` からテーマに合う施設を抽出。

**選定プロセス:**
1. テーマの条件でフィルタリング（prefecture, coupleOk, priceMin, features 等）
2. 選定基準を3-4つ明確にする（駅チカ/カップルOK/初心者向き 等）
3. 5施設を選定（エリア・価格帯・体験価値のバランスを考慮）

### 3. 裏どり（必須）

**全施設の公式サイトをPlaywright or WebFetchで確認する。以下を検証:**

| チェック項目 | 確認方法 |
|------------|---------|
| カップル/男女利用OK | 公式サイトに「男女利用可」「カップルOK」の明記があるか |
| 完全個室か | 「個室」「プライベート」等の記載 |
| 料金が正しいか | 公式の料金表と `facilities.json` を突き合わせ |
| 営業中か | 閉店・休業情報がないか |
| 駅・アクセス | 最寄り駅と徒歩分数が正しいか |

**裏どりNGの場合:**
- 公式サイトで確認できない情報は投稿に含めない
- 男女利用の明記がない施設は「デート向き」として紹介しない
- データ乖離があれば `facilities.json` を修正してからデプロイ

**裏どり結果をユーザーに報告し、OKをもらってから画像作成に進む。**

### 4. データ乖離修正

裏どりで見つかった `facilities.json` との差異を修正:
1. 該当フィールドを Edit で修正
2. description / saunakoCommentLong に矛盾があれば修正
3. `npm run build` でビルド確認
4. コミット & プッシュ

### 5. 画像作成

`scripts/x-list-card.html` をテンプレートとして使用。

**画像に含める情報（各施設）:**
- 施設名
- 最寄り駅 / 徒歩分数
- 価格 / 利用時間
- ひとこと推しポイント（テーマの文脈に合わせた1行）

**画像のヘッダー:**
- カテゴリラベル（例: PRIVATE SAUNA × DATE）
- タイトル（例: 東京で初デート向きの個室サウナ 5選）
- 選定基準タグ（3-4個）

**画像のフッター:**
- 情報時点（例: 2026年3月時点の情報です）
- saunako.jp

**生成手順:**
1. `scripts/x-list-card.html` のタイトル・施設情報・タグを Edit で書き換え
2. ローカルHTTPサーバーを起動（`python3 -m http.server 8765`）
3. Playwrightでビューポート 1080x1350 に設定
4. `http://localhost:8765/x-list-card.html` にアクセス
5. スクリーンショットを撮影（png）
6. サーバー停止
7. `open` コマンドでプレビュー表示 → ユーザー確認

### 6. 投稿文作成

**構成:**
```
{1行目: テーマを端的に}

選んだ基準は
・{基準1}
・{基準2}
・{基準3}
・{基準4}

{保存理由・使いどころ}

#{ハッシュタグ1} #{ハッシュタグ2}
```

**ルール:**
- 280ウェイト以内（日本語1文字=2, ASCII=1, 絵文字=2, 改行=1）
- 本文にURLを入れない
- ハッシュタグは2つまで
- 最後の1行で「保存したくなる理由」を入れる
- サウナ子の友達口調（docs/character-guide.md 準拠）

### 7. ユーザー確認（必須 — スキップ禁止）

**投稿前に必ずユーザーの承認を得る。自動投稿しない。**

以下をすべて表示し、AskUserQuestion で確認:

1. **選定した施設一覧**（テーブル形式: 名前・駅・価格・推しポイント）
2. **裏どり結果**（問題なし or 修正済み）
3. **画像プレビュー**（`open` コマンドでmacOSプレビューを起動。ユーザーが実際に目で見て確認する）
4. **投稿文**（コードブロックでそのまま表示）

**ユーザーが画像と投稿文の両方を確認し、明示的に「OK」「投稿して」等と言うまで投稿に進まない。**
修正指示があれば画像/テキストを修正して再度確認。

### 8. 投稿

**ユーザーの明示的な承認後のみ実行:**

1. Playwrightで `https://x.com/compose/post` を開く
2. 「Add photos or video」→ 画像ファイルをアップロード
3. テキスト入力（`slowly: true` で入力）
4. Postボタンをクリック（`evaluate` でJS直接実行）
5. Tweet IDを記録

### 9. スレッド補足（オプション）

ユーザーが希望する場合のみ:
- 2ツイート目: 選定理由の補足
- 3ツイート目: saunako.jp への自然な導線（「条件で探したい人はプロフのリンクからどうぞ」）
- `reply_to_tweet_id` で本文に返信する形で投稿

### 10. ログ記録

`docs/x-post-log.md` に追記:

```
| 日付 | カテゴリ | テーマ | 施設IDs | データ修正 | 投稿済み |
```

## デザインテンプレート

`scripts/x-list-card.html` が画像テンプレート。
- サイズ: 1080x1350px
- 配色: ネイビー(#0f1724〜#1a2844) × オレンジ(#E87957)
- フォント: Hiragino Kaku Gothic ProN

タイトル・施設・タグ部分を Edit で書き換えて使う。

## 注意事項

- **裏どりなしで投稿しない**。全施設の公式サイトを必ず確認
- 価格は公式サイトの最新情報を反映（オフピーク/ピークの注記があれば明記）
- 「カップルOK」は公式サイトに明記がある施設のみ
- 閉店・休業施設は除外
- データ修正があればビルド確認 + コミット + プッシュしてから投稿
- 施設への否定的な情報は含めない
