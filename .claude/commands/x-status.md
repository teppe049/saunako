# X ステータスチェック

@saunako_jp の現在の状況をPlaywrightで実測し、簡潔にレポートする。

引数: $ARGUMENTS（`簡易` / `詳細` / 省略時は `簡易`）

## ツール選択

**Playwright MCP をサブエージェントで実行**（メモリやdocsの数値を信用しない。必ず実測）。

## 共通: アカウント基本情報の取得

サブエージェント（Sonnet）で以下を実行:

1. `browser_navigate` で `https://x.com/saunako_jp` にアクセス
2. `browser_snapshot` でプロフィール情報を取得:
   - フォロワー数 / フォロー数 / 投稿数
3. 最新5件の投稿を確認:
   - 投稿内容（要約）/ 日時 / いいね数 / RT数 / リプライ数
   - 画像の有無
4. ピン留めツイートのエンゲージメント

## モード: 簡易（デフォルト）

### 出力フォーマット

```
## X ステータス（M/D 実測）

### アカウント
| 項目 | 数値 | 前回 | 変化 |
|------|------|------|------|
| フォロワー | XX人 | XX人 | +X |
| フォロー | XX人 | — | — |
| 投稿数 | XX件 | — | — |

### 直近5投稿
| # | 日時 | 内容（要約） | ♥ | RT | 💬 |
|---|------|------------|---|----|----|
| 1 | M/D HH:mm | ○○○○ | X | X | X |

### ピン留め
♥ XX / RT X / Bookmark X

### 所感
- {フォロワー推移の評価}
- {エンゲージメントの傾向}
- {次のアクション提案}
```

### 前回データとの比較

メモリ `MEMORY.md` の「X戦略」セクションにあるフォロワー数と比較する。
**ただしメモリの数値は参考値** — 実測値を正とする。

### メモリ更新

実測後、`MEMORY.md` の該当行を最新のフォロワー数・実測日に更新する。

## モード: 詳細

簡易モードに加えて以下を追加取得:

### 追加取得（サブエージェントで実行）

1. **直近10投稿**のエンゲージメント（簡易は5件）
2. **フォロワーリスト**: `x.com/saunako_jp/followers` を開いて最新フォロワーを確認
3. **通知**: `x.com/notifications` でリプライ・メンション・いいね通知を確認（ログイン済みの場合のみ）

### 追加出力

```
### エンゲージメント分析
- 平均いいね: X.X / 投稿
- 最高いいね投稿: {内容要約}（♥ XX）
- 投稿頻度: X件/週

### 最新フォロワー（直近5人）
| # | アカウント | フォロワー数 | bio要約 |
|---|-----------|------------|--------|

### 週次サマリー
- 今週の投稿数: X件
- フォロワー増減: +X人
- 目標(100人)達成率: XX%
- 100人到達予測: {現ペースでの予測}
```

## 注意事項

- Playwright MCP は**必ずサブエージェント（Sonnet）で実行**する
- Views はログインなしでは取得不可 — 取れない場合は「N/A」で出力し、無理に推測しない
- GitHub Actions の自動投稿が失敗していないか `docs/x-drafts.md` の状態も軽く確認する
- 実測で得た数値は信頼できるが、Guest Token の制限で取れない指標がある旨を注記する
