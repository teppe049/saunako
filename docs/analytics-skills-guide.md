# サウナ子運営者のためのアナリティクス判断力ガイド

AI がデータ収集・実装を担当する時代に、人間が磨くべきスキルは「操作」ではなく「判断」。
このドキュメントは、saunako の運営を通じて身につけるべき判断力を整理したもの。

---

## 1. 数字の裏にある「人の行動」を想像する

GA4 や Search Console の数字は、すべて人間の行動の結果。
数字を見たとき「なぜそうなったのか」を想像する癖をつける。

### 実践例

| 数字 | 表面的な読み方 | 行動を想像した読み方 |
|------|--------------|-------------------|
| 週末にPVスパイク（土日で全体の45%） | 「週末にアクセスが多い」 | ユーザーは金曜夜〜土曜に「今週末どこ行こう」と検索している。**金曜にSNS投稿や記事公開すると効果的** |
| 滞在時間 536秒の日がある | 「長く見てる人がいる」 | 誰かが複数施設をガッツリ比較検討している。**検索→詳細→比較の導線が機能している証拠** |
| Direct比率 6%（下降トレンド） | 「直接流入が減った」 | 「saunako」と指名検索する人が少ない = ブランド想起が弱い。**Organic は伸びているが、リピーターがつくフェーズにはまだ到達していない** |
| ある施設だけ直帰率 60%超 | 「ページが悪い」 | その施設ページに来た人が期待した情報がなかった。**料金？写真？予約リンク？何が足りないのか仮説を立てて検証** |
| モバイル比率 73% | 「スマホユーザーが多い」 | 移動中や外出先で「近くの個室サウナ」を探している。**現在地検索・地図機能の重要度が高い** |

### コツ

- 「増えた/減った」で止まらない。「**だから何をすべきか**」まで考える
- 数字が動いた日に何があったかを思い出す（施策を打った？曜日の影響？天気？）
- 1日単位の変動に一喜一憂しない。**週単位のトレンド**を見る

## 2. 「なぜこの数字なのか」の仮説を立てる

AI は「imp が高くて CTR が低い」というパターンは見つけられる。
でも **なぜクリックされないのか** の本質的な理由は、サービスとユーザーを理解している人間にしか分からない。

### saunako での実例

**高輪SAUNAS の料金問題**（2026/3/10 発見）

- AI の分析: 「124imp / CTR 2.4%。seoDescription を追加すれば改善する」
- 人間の判断: 「待って、この料金は共用サウナの値段で、個室サウナの料金じゃない。誤解を招く」

→ データだけ見ていたら気づかない。**施設の実態を知っている人間だからこそ出せた判断**。

### 仮説立案のフレームワーク

```
1. 数字を見る     → 「CTR が 0% だ」
2. なぜ？を問う   → 「検索結果に表示されてるのにクリックされない理由は？」
3. 仮説を立てる   → 「title にユーザーが求める情報（料金？口コミ？）が入っていないのでは」
4. 検証する       → title を変えて 1-2 週間後に再計測
5. 学びを蓄積する → 「料金を入れるとCTRが上がる傾向がある」
```

大事なのは **仮説が間違っていてもいい** ということ。間違った仮説から学べることの方が多い。

## 3. 「何をやらないか」を決める

Issue は無限に増える。AI に聞けば「やった方がいい」ことは 100 個出てくる。
しかしリソース（時間・お金・集中力）は有限。**やらないことを決める力**が最も価値が高い。

### 優先順位の判断軸

```
         効果大
          │
    P1    │    P0
  (今週)   │  (今すぐ)
          │
──────────┼──────────
          │
    P3    │    P2
 (バックログ) │  (来週)
          │
         効果小
     工数大 ←──→ 工数小
```

### 判断の実践例（2026/3/10 のセッション）

30 個の open Issue から以下のように絞り込んだ:

| 判断 | 理由 |
|------|------|
| **#131（送客計測）を最優先にした** | マネタイズの土台。データが溜まるまで時間がかかるので、早く仕込むほど後で効く |
| **#129（SAUNA汽汽）を今週にした** | 4月オープン前に掲載 = 指名検索を先取りできる。時間制約がある |
| **パフォーマンス系（#123, #128）を後回しにした** | 現状問題ないものを最適化しても成長に直結しない |
| **A/Bテスト基盤（#112）を見送った** | トラフィックが少ない段階で A/B テストしても統計的に有意な結果が出ない |

### 「やらない」判断のコツ

- 「面白そう」と「今やるべき」は違う
- 「完璧にやる」より「70点で出して反応を見る」方が早く学べる
- 迷ったら「これをやらなかったら何が起きるか？」と問う。何も起きないならやらなくていい

## 4. ユーザー視点で判断する

技術的に正しいことと、ユーザーにとって正しいことは違う。

### 考えるべき問い

- **このページに来た人は、何を期待している？**
  - 「個室サウナ 料金」で来た人 → 料金比較表がないとすぐ離脱する
  - 「〇〇サウナ 口コミ」で来た人 → 実体験の情報がないと物足りない

- **この変更で、ユーザーの体験はどう変わる？**
  - 送客計測基盤（/go/ リダイレクト）→ ユーザーに影響なし → OK
  - もし「予約にはログインが必要」にしたら → 離脱する → NG

- **自分がユーザーだったら、このサイトで何をする？**
  - 実際に saunako で施設を探してみる。どこで迷う？何が足りない？

### 「ユーザーの声」の代替手段

レビューや問い合わせがない段階では、以下がユーザーの声の代わりになる:

| データ | ユーザーの声として読むと |
|--------|---------------------|
| 直帰率が高いページ | 「期待した情報がなかった」 |
| 滞在時間が長いページ | 「じっくり読む価値があった」 |
| 検索クエリ「〇〇 料金」 | 「料金が知りたい」 |
| 検索クエリ「〇〇 口コミ」 | 「実際どうなのか知りたい」 |
| /search ページのPVが多い | 「もっと絞り込みたい」 |

## 5. 競合・市場を見る目を持つ

データだけ見ていると自サイトの最適化にしか目が向かない。
でも検索順位は **相対的なもの** — 競合が何をしているかで自分の順位は変わる。

### 定期的にチェックすべきこと

- **「個室サウナ」で検索して上位 10 サイトを見る**
  - どんなコンテンツがある？
  - 自分にあって相手にないものは？（= 差別化ポイント）
  - 相手にあって自分にないものは？（= 改善ポイント）

- **新しい競合の出現を監視する**
  - 突然順位が下がったら、新しいサイトが参入した可能性がある

- **施設オーナーの視点で考える**
  - 「saunako に掲載する価値は何か？」→ 送客データで示す
  - 「他の掲載サイトと何が違う？」→ 393施設の網羅性、比較機能

### 競合分析のフレームワーク

```
1. 「個室サウナ」で Google 検索
2. 1ページ目の全サイトを開く
3. 以下を記録:
   - コンテンツの種類（記事？一覧？比較？）
   - 施設数
   - UI/UX の特徴
   - 独自の機能
4. saunako との差分を整理
5. 「真似すべきこと」と「差別化すべきこと」を分ける
```

## 6. 「いつ判断するか」のリズムを作る

データは毎日見ても意味がない。判断のリズムを決めておく。

### 推奨サイクル

| 頻度 | やること | 所要時間 |
|------|---------|:--------:|
| **毎週金曜** | GA4 で週間 PV/UU をざっと確認。大きな変動がないか | 5分 |
| **隔週** | `/analytics-pdca` で SC データと突き合わせ。Quick Win を特定 | 30分 |
| **月1回** | 競合チェック + Issue の優先度見直し + 翌月の方針決め | 1時間 |
| **四半期** | KPI 振り返り + 中期方針の見直し | 2時間 |

### 判断のトリガー

定期サイクル以外に、以下のタイミングでは即判断が必要:

- PV が突然 2倍以上になった → バズ？何が起きた？次のアクションは？
- 検索順位が急落した → Google アルゴリズム更新？競合の動き？
- 新しい施設オーナーから問い合わせが来た → マネタイズの機会

## 7. 失敗から学ぶ仕組みを持つ

SEO 施策は「やってみないと分からない」ことが多い。大事なのは **失敗を記録して次に活かす** こと。

### saunako の施策ログから見える学び

| 施策 | 結果 | 学び |
|------|------|------|
| 24h記事 title に「2026年最新」追加 | pos6.5 のまま CTR 0% | title だけ変えてもダメ。**コンテンツ自体の改善が必要** |
| 埼玉エリア title に「カップルOK」追加 | pos3 / CTR 67% | ニッチなエリア × 具体的な条件は **即効性あり** |
| くつろぎ desc に地域名追加 | CTR 3.6% → 4.4% | **地域名を入れると指名検索の CTR が上がる** |

### 記録すべきこと

```
- 何をやったか（施策の内容）
- なぜやったか（仮説）
- どうなったか（結果の数字）
- なぜそうなったか（考察）
- 次にどう活かすか（学び）
```

→ これが MEMORY.md の施策ログの役割。AI に計測させて、人間が「なぜ」を考える。

## まとめ: AI 時代の運営者に必要な 7 つの力

| # | スキル | AI にできること | 人間がやるべきこと |
|---|--------|---------------|-----------------|
| 1 | 行動を想像する力 | 数字を出す | 数字の裏にある行動を読む |
| 2 | 仮説を立てる力 | パターンを見つける | なぜそうなるかの理由を考える |
| 3 | 捨てる力 | 選択肢を100個出す | 5個に絞る |
| 4 | ユーザー視点 | 技術的に正しいものを作る | ユーザーにとって正しいか判断する |
| 5 | 市場を見る目 | 自サイトのデータを分析する | 競合・市場の文脈で解釈する |
| 6 | リズムを作る力 | いつでもデータを出せる | いつ見るか、いつ判断するかを決める |
| 7 | 失敗から学ぶ力 | 結果を記録する | なぜ失敗したかを考え、次に活かす |

**AI は「作業」を担当し、人間は「判断」を担当する。**
この分業がうまく回ると、一人でも大きなサービスを運営できる。
