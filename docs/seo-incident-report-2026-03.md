# SEOインシデントレポート: Google検索トラフィック消滅事件（2026年3月）

## 何が起きたか

2026年3月11日を境に、saunako.jp の Google 検索トラフィックが**完全にゼロ**になった。

表示回数（impression）がゼロ。クリック数もゼロ。インデックスはされている。手動ペナルティもない。ただ、Google検索結果に一切表示されなくなった。

一方で、Bing では引き続き好調だった。誰もそのことに気づいていなかった。

---

## タイムライン

### 成長期（2/20〜3/9）

saunako.jp は2月下旬からGoogle検索で急成長していた。

| 日付 | 表示回数 | クリック数 |
|------|:-------:|:--------:|
| 2/20 | 3 | 1 |
| 2/28 | 278 | 8 |
| 3/5 | 977 | 16 |
| 3/8 | 740 | 24 |
| 3/9 | 706 | 20 |

2週間でimpが3→977に爆増。サイトが軌道に乗り始めた時期だった。

### 崩壊（3/10〜3/11）

| 日付 | 表示回数 | クリック数 |
|------|:-------:|:--------:|
| 3/10 | 250 | 3 |
| 3/11 | 0 | 0 |
| 3/12〜3/24 | 0 | 0 |

3/10に急落し、3/11以降は完全にゼロ。以後2週間以上、回復の兆しなし。

### 発見（3/27）

`/analytics-pdca` の定期分析で発見。それまで2週間以上、異変に気づかなかった。

気づかなかった理由は後述する。

---

## なぜ気づかなかったのか

### GA4のOrganic Searchを信じていた

GA4のチャネルレポートでは、Organic Search は週51PVで「42%のシェアで安定」と見えていた。だから問題に気づかなかった。

しかし内訳を分解すると:

| 検索エンジン | PV | 比率 |
|-------------|:--:|:----:|
| Bing | 46 | 90% |
| Ecosia（Bing系） | 4 | 8% |
| Google | 1 | 2% |

**Organic Search の 90% は Bing だった。** GA4のチャネルレポートは検索エンジンを区別しない。「Organic Search 42%」という数字の裏側で、Googleからのトラフィックはほぼ消滅していた。

### Search Console だけを見ていた

Search Console は Google の検索データしか表示しない。Bingの好調さはSCには一切現れない。

SCで「CTR 0%」の施設を見つけるたびに seoTitle や seoDescription を書き換えていた。それがさらに状況を悪化させていたかもしれない。Bingでは問題なく表示されていたページのタイトルを、Google SC の数字だけを根拠に何度も変更していた。

---

## 何が原因だったのか

### 直接原因: 128施設の seoTitle 一括変更

3/10前後に、施設ページのSEO改善として以下をデプロイした:

```
feat(seo): 関東128施設にseoTitle一括設定
feat(seo): 17施設にseoTitle一括設定（Tier1+Tier2）
feat(seo): Quick Wins 16施設にseoTitle一括設定
feat(seo): SEO全施策11件を一括実装
```

合計160施設以上のタイトルを、数日間で一気に変更した。

Googleから見れば、あるサイトが突然160ページのタイトルを書き換えたように見える。しかも Claude が生成した、似たようなフォーマットのタイトルを一括適用。これは「SEO操作」と判断されても不思議ではない。

### 背景要因: Google アルゴリズムの大変動期

2月14日頃から、Googleは未公表の大規模ランキング変動を起こしていた。SEMrush Sensorの変動スコアは9.4/10（通常は2-4程度）。55%以上のサイトが順位変動を経験した。

アルゴリズムが不安定な時期に、サイト側も大規模な変更を加えた。最悪のタイミングだった。

### 構造的要因: AI生成コンテンツの品質評価

saunako.jp の施設説明、記事、seoTitle/seoDescription の大半は Claude で生成されている。Google は E-E-A-T（Experience, Expertise, Authoritativeness, Trustworthiness）を重視しており、特に「Experience（実体験）」の評価を強化している。

テンプレ的なAI生成コンテンツが大量にあるサイトは、品質フィルターに引っかかりやすい。393施設のページが似たような構造・似たような文体で書かれていれば、「人間が関与していないサイト」と判断されるリスクがある。

## 教訓

### 1. SEOメタデータの一括変更は絶対にやらない

「効率的に全部やろう」は最大のリスク。title/description の変更は1回のデプロイにつき最大10ページ。1-2週間あけて効果を確認してから次のバッチ。

これは Hook（hookify）で自動ブロックするようにした。

### 2. GA4 の Organic Search は検索エンジン別に分解する

「Organic Search 42%」の中身を見ないと、どの検索エンジンが貢献しているか分からない。GA4で `sessionSource` を分解すれば、google / bing / yahoo を個別に把握できる。

これを知らなかったせいで、Googleのトラフィック消滅に2週間気づかなかった。

### 3. Search Console の CTR 0% で即座に title 変更に走らない

SC で CTR 0% のページがあっても、それは「Googleで表示されてクリックされていない」だけ。Bing や他の検索エンジンでは上位表示されているかもしれない。

まず GA4 で `sessionSource` を確認し、本当にどの検索エンジンからも来ていないのか確認する。Bingで好調なページの title を変えると、Bing の順位まで失うリスクがある。

### 4. AI生成サイトは E-E-A-T を意識的に入れる

全ページをAIで生成すると、Googleの品質フィルターに引っかかりやすい。対策:

- 人間の実体験を混ぜる（訪問レポート、オーナー取材）
- データ分析記事（DBを持っている人にしか書けない一次情報）
- 著者情報・運営者情報の充実
- テンプレ感の排除（施設ごとに異なる切り口で書く）

全施設に行く必要はない。「393施設を徹底調査したキュレーター」というポジションで、調査の深さ自体が Experience になる。

### 5. アルゴリズム変動期に大規模変更しない

Google のランキングが不安定な時期に、サイト側も大きく変えると、変動の影響と変更の影響が重なって最悪の結果になる。大規模SEO施策の前に「Google algorithm update」で検索して、変動期でないか確認する。

### 6. Bing Webmaster Tools を最初から登録する

Bingのシェアは無視できない。saunako.jp では Google の20倍のクリック数があった。Copilot経由の検索も増えており、今後さらに重要になる。Google Search Console からインポートすれば数分で完了するので、サイト立ち上げ時に必ず登録する。
