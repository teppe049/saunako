# SEO全施策 検証チェックリスト

2026-03-14 実装分（11施策）のデプロイ後検証項目。

## Phase 1: 構造化データ・基盤強化

### 1-A: 検索ページ ItemList JSON-LD
- [ ] `/search` のソースに `ItemList` JSON-LD が出力されている
- [ ] `numberOfItems` が検索結果件数と一致
- [ ] `itemListElement` が先頭10件のみ（11件以上にならない）
- [ ] [Rich Results Test](https://search.google.com/test/rich-results) でエラーなし

### 1-B: パンくず4階層化
- [ ] エリア登録済み施設（例: `/facilities/1`）でパンくずが `TOP > 東京都 > 新宿・港・銀座 > 施設名` の4階層
- [ ] エリア未登録施設で3階層にフォールバック
- [ ] JSON-LD `BreadcrumbList` の `position` が連番（1,2,3,4）
- [ ] パンくずのエリアリンクが正しいサブエリアページに遷移
- [ ] Rich Results Test でパンくずが認識される

### 1-C: LocalBusiness 詳細化
- [ ] `priceRange` が `1名60分 ¥3,000〜` 形式になっている（旧: `¥3,000〜`）
- [ ] 営業時間が `10:00〜22:00` パターンの施設で `openingHoursSpecification` が出力
- [ ] パース不能な営業時間（例: `不定期`）の施設で `openingHoursSpecification` が出力されない
- [ ] `openingHours`（文字列版）も引き続き出力されている

### 1-D: 画像alt最適化
- [ ] 施設詳細のメイン画像altが `施設名 個室サウナ 特徴` 形式
- [ ] 検索結果カードの画像altが `施設名 個室サウナ`
- [ ] 関連施設カードの画像altが `施設名 個室サウナ`
- [ ] ライトボックス画像のaltも最適化されている

## Phase 2: UX・内部リンク強化

### 2-A: フッター条件別フィルタリンク
- [ ] モバイル: フッターに「条件から探す」セクションが2カラムで表示
- [ ] PC: フッターに「条件から探す」が横並びで表示
- [ ] 6リンク全てが正しい検索URLに遷移: `coupleOk=true`, `waterBath=true`, `selfLoyly=true`, `outdoorAir=true`, `priceMax=3000`, `priceMax=5000`
- [ ] リンク先で検索結果が正しくフィルタされる

### 2-B: 近隣施設の距離表示
- [ ] 関連施設（同エリア）カードに `ここから○km` が表示される
- [ ] 両施設に座標がある場合のみ表示（座標なしは距離非表示）
- [ ] 類似価格帯カードにも `（○km）` が表示される
- [ ] 距離の値が妥当（同エリアなら概ね10km以内）

### 2-C: 画像サイトマップ
- [ ] `/sitemap-images.xml` にアクセスして有効なXMLが返る
- [ ] 各施設URLに `<image:image>` タグが含まれる
- [ ] `<image:title>` が `施設名 個室サウナ` 形式
- [ ] 閉店施設が含まれていない
- [ ] `/robots.txt` に `sitemap-images.xml` のURLが記載されている
- [ ] Google Search Console でサイトマップを送信して正常に処理される

### 2-D: GEO対策（1行要約）
- [ ] h1直下に `{都道府県}{市区町村}にある個室サウナ。` が表示
- [ ] 駅情報がある施設: `○○駅から徒歩○分。` が含まれる
- [ ] 料金がある施設: `料金○,○○○円〜。` が含まれる
- [ ] 旧来の駅情報表示（独立行）が削除され重複がない
